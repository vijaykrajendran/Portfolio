# Installing Nginx on Linux

Let's get Nginx up and running on your Linux machine. I'll show you the easiest way.

## Prerequisites

- Ubuntu 20.04+ or Debian 10+
- `sudo` access on your machine
- Basic terminal knowledge

## Step 1: Update Your System

Before installing anything new, always update your system:

```bash
sudo apt update
sudo apt upgrade -y
```

## Step 2: Install Nginx

The simplest wayâ€”install from the default repository:

```bash
sudo apt install nginx -y
```

This installs Nginx with default modules. If you need specific modules later, we'll cover that.

**Verify installation:**

```bash
nginx -v
# Output: nginx version: nginx/1.18.0 (Ubuntu)
```

## Step 3: Start Nginx Service

Start the Nginx service and enable it to run on boot:

```bash
sudo systemctl start nginx
sudo systemctl enable nginx
```

**Check status:**

```bash
sudo systemctl status nginx
```

You should see:
```
â— nginx.service - A high performance web server and a reverse proxy server
     Loaded: loaded (/lib/systemd/system/nginx.service; enabled; vendor preset: enabled)
     Active: active (running)
```

## Step 4: Verify Nginx is Running

Open your browser and visit `http://localhost`:

```
You should see the Nginx welcome page!
```

Or use curl:

```bash
curl http://localhost
```

## Step 6: Basic Commands

```bash
# Check if config is valid
sudo nginx -t

# Reload config (graceful, no downtime)
sudo systemctl reload nginx

# Restart Nginx
sudo systemctl restart nginx

# Stop Nginx
sudo systemctl stop nginx

# Start Nginx
sudo systemctl start nginx

# View Nginx processes
ps aux | grep nginx
```

## Step 7: Your First Configuration

Let's look at the main config file:

```bash
sudo nano /etc/nginx/nginx.conf
```

You'll see something like:

```nginx
user www-data;
worker_processes auto;
pid /run/nginx.pid;

events {
    worker_connections 768;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    # ... more settings ...

    include /etc/nginx/sites-enabled/*;
}
```

**Key lines:**
- `user www-data` â†’ Nginx runs as this user
- `worker_processes auto` â†’ Use all CPU cores
- `worker_connections 768` â†’ Max connections per worker
- `include /etc/nginx/sites-enabled/*` â†’ Load all enabled sites

## Step 8: Test Configuration Changes

Always validate before applying:

```bash
sudo nginx -t
```

Output:
```
nginx: the configuration file /etc/nginx/nginx.conf syntax is ok
nginx: configuration file /etc/nginx/nginx.conf test is successful
```

Then reload:

```bash
sudo systemctl reload nginx
```

## Troubleshooting

### Nginx won't start
```bash
# Check error logs
sudo tail -f /var/log/nginx/error.log

# Validate config
sudo nginx -t
```

### Port 80 already in use
```bash
# Find what's using port 80
sudo lsof -i :80

# Or kill the process
sudo fuser -k 80/tcp
```

### Permission denied errors
```bash
# Run as sudo
sudo systemctl restart nginx

# Or check file permissions
ls -la /etc/nginx/nginx.conf
```
